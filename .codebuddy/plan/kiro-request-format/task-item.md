# Kiro 渠道请求参数格式实现任务清单

## 项目概述

实现Kiro（AWS CodeWhisperer）渠道的原生请求参数格式支持，包括单轮对话、多轮对话、工具调用、流式响应等多种场景的完整格式规范。

## 实施计划

### 阶段一：核心数据结构定义（基础框架）

- [ ] **1. 定义核心请求/响应数据结构**
  - 创建`kiro_request.go`文件，定义`KiroRequest`结构体
  - 实现`ConversationState`结构体，包含聊天触发类型、对话ID、当前消息
  - 定义`CurrentMessage`结构体，支持内容和用户输入上下文
  - 实现`UserInputMessageContext`结构体，包含工具定义和工具结果
  - 创建`kiro_response.go`文件，定义`KiroResponse`结构体
  - 实现`AssistantResponseMessage`结构体，支持内容和工具调用
  - 定义`Usage`结构体，用于Token使用统计
  - _需求：基础请求格式、通用响应结构_

- [ ] **1.1 实现工具定义结构**
  - 创建`kiro_tools.go`文件，定义`ToolSpecification`结构体
  - 实现`InputSchema`结构体，支持JSON Schema格式
  - 定义`ToolCall`结构体，包含工具调用ID、名称和输入参数
  - 实现`ToolResult`结构体，用于工具执行结果
  - _需求：工具调用相关结构定义_

- [ ] **1.2 实现对话历史管理**
  - 创建`kiro_history.go`文件，定义`ConversationHistory`结构体
  - 实现`UserInputMessage`和`AssistantResponseMessage`历史记录结构
  - 支持历史消息的顺序管理和时间戳记录
  - _需求：多轮对话历史格式_

### 阶段二：协议转换层实现

- [ ] **2. 实现内部格式到Kiro原生格式转换**
  - 创建`kiro_converter.go`文件，实现`ToKiroRequest()`方法
  - 将内部聊天消息格式转换为Kiro原生请求格式
  - 处理模型ID、配置ARN等必需字段的填充
  - 实现对话状态的序列化和反序列化
  - _需求：所有场景的请求格式转换_

- [ ] **2.1 实现Kiro响应到内部格式转换**
  - 在`kiro_converter.go`中实现`FromKiroResponse()`方法
  - 解析Kiro原生响应，提取助手回复内容
  - 处理工具调用请求和结果
  - 转换使用统计信息
  - _需求：响应格式解析和处理_

- [ ] **2.2 实现工具调用转换**
  - 创建`kiro_tool_converter.go`文件
  - 实现内部工具定义到Kiro工具格式的转换
  - 处理工具输入Schema的映射和验证
  - 支持工具调用结果的解析和转换
  - _需求：工具调用相关转换逻辑_

### 阶段三：对话状态管理

- [ ] **3. 实现对话状态管理器**
  - 创建`kiro_conversation_manager.go`文件
  - 实现对话ID的生成和管理
  - 支持多轮对话历史的维护和更新
  - 处理对话上下文的序列化和持久化
  - _需求：多轮对话状态管理_

- [ ] **3.1 实现历史消息处理**
  - 在对话管理器中实现历史消息的添加和修剪
  - 支持最大历史消息长度的配置
  - 处理消息时间戳和顺序管理
  - 实现历史消息的压缩和优化
  - _需求：对话历史格式处理_

### 阶段四：工具调用处理

- [ ] **4. 实现工具调用处理器**

  - 支持多个工具的同时调用
  - 处理工具调用ID的生成和匹配
  - _需求：工具调用请求处理_

- [ ] **4.1 实现工具结果集成**

  - 支持工具结果的格式转换和验证
  - 处理工具调用错误和异常情况
  - 实现工具结果的缓存和重用
  - _需求：工具执行结果处理_

### 阶段五：流式响应处理

- [ ] **5. 实现流式响应解析器**
